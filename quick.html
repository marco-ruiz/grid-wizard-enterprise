Getting Started

The domain problem GWE addresses includes some specialized terminology and concepts (industry and proprietary); which are necessary to be familiarized in order to get the most out of a GWE user's experience. To facilitate this process, an introductory glossary page has been included in this site with the most important concepts and terminology. We strongly recommend users to review it before start working with GWE.

Getting GWE setup is simple! Just follow the steps detailed in the Quick Start Guide .

Please send us your questions, ideas and/or feedback at gwe-support@nbirn.net or share them with the entire project community in the project forum . Finally, don't forget to join the GWE user's mailing list to receive updates on the project's progress.

Thanks for your support. Enjoy!

Quick Start Guide

  • Install GWE in your machine.

    Unbundle one of the GWE bundles and set the appropriate environmental variables. Check the 'Installation Guide' for details.

  • Configure installed GWE.

    Provide the authentication information to access networked remote resources (file systems and clusters) and the list of clusters to composed your "GWE Computational Grid". Check the 'Configuration Guide' for details.

  • Run the 'GWE Daemon Remote Installer' utility.

    You can skip this step if you have access to a 'GWE enabled' cluster. Promoting a standard cluster to the 'GWE enabled' level is as simple as invoking the 'GWE Daemon Remote Installer'. Check the 'GWE Administrator's Guide' for details.

  • Launch GWE client.

    GWE comes with a modular API to build client applications on top of it. Out of the box, GWE includes 2 client applications ready to use: GWE Terminal and GWE Commands. Check the 'GWE User's Guide' for details.

  • Interact with your 'GWE enabled' cluster!

    That's it! You are ready to interact with a GWE enabled cluster (submit process parallelization requests, monitor execution progress, control orders execution, etc). To queue jobs into your GWE cluster you need to describe them using P2EL. Check the 'P2EL Specification' for details.
